# ADR 0009: Initiale Lagerorte

## Status

accepted

## Kontext

Freie und blockierte Sets sowie einzelne Komponenten muessen auffindbar gelagert werden. Beim Vervollstaendigen von Sets soll die App passende Komponenten inklusive Lagerort vorschlagen.

## Entscheidung

Zum Start werden folgende Lagerorte angelegt:

| Name | Typ | Kapazität | Plätze |
| --- | --- | ---: | --- |
| W1 | iPadwagen | 30 | 1-30 |
| W2 | iPadwagen | 30 | 1-30 |
| W3 | iPadwagen | 30 | 1-30 |
| W4 | iPadwagen | 30 | 1-30 |
| W5 | iPadwagen | 30 | 1-30 |
| W6 | iPadwagen | 30 | 1-30 |
| Schrank1 | Schrank |  | keine festen Plätze |
| Schrank2 | Schrank |  | keine festen Plätze |
| Regal1 | Regal |  | keine festen Plätze |

## Konsequenzen

- Lagerorte koennen als Seed-Daten angelegt werden.
- iPadwagen haben eine Kapazität von je 30 Plätzen.
- Bei iPadwagen wird der konkrete Platz dokumentiert.
- Wagenplätze sollen vorrangig mit kompletten Sets belegt werden.
- Einzelkomponenten koennen gelagert werden, sollen aber nicht die bevorzugte Belegung von Wagenplätzen sein.
- Weitere Lagerorte koennen spaeter ergaenzt werden.
